# 知床財団オリジナル商品 売上・在庫分析 ＆ 発注計画システム

本フォルダは、知床（Shiretoko）ブランドにおけるオリジナル商品の過去売上実績（2年間）、現在の店舗在庫、および取引先（知床財団）の在庫データを統合し、最適な仕入発注計画を策定するためのシステムとデータ群が格納されています。

---

## 📂 主要ファイル構成

### 1. 実行スクリプト
* **`prepare_order.py`**: 発注計画算出のメインスクリプト。実績値から指定消化期間（7ヶ月間）の目標販売数を割り出し、店舗在庫および取引先在庫を引き当てて最終発注計画を作成します。
* **`add_sku_comparison_sheet.py`**: `シレトコ分析.xlsx` 内に「品番数比較」シートを自動生成・更新するスクリプト。年度別の売上比較に加え、店舗現在庫と取引先在庫を並べて一覧化します。

### 2. インプットデータ（入力ファイル）
* **`シレトコ分析.xlsx`**: 今年度の売上トランザクションデータ（`営業日付別売上分析(旧)` シート）。
* **`営業日付別売上分析(旧) (1).csv`**: 昨年度の売上トランザクションデータ（シフト変換・Excelシリアル値対応済）。
* **`在庫一覧 .csv`**: 自社店舗の最新在庫データ（Shift-JIS/CP932）。
* **`知床財団在庫表260708.xlsx`**: 取引先（知床財団）から共有された最新の在庫表。

### 3. アウトプットデータ（出力成果物）
* **`Shiretoko_発注計画.xlsx`**: 自動生成される発注計画Excel。
  - **店舗別発注まとめ**: 各店舗へ割り振るべき目標数・現在庫・発注数。
  - **商品別発注まとめ**: 取引先在庫との対比（確定発注数と取引先欠品数の算出）。
  - **発注詳細データ**: 店舗×SKU別の詳細な計算データ。

---

## ⚙️ 計算ロジックとルール

本システムは、無駄な滞留在庫を防ぎつつ、売れ筋商品の欠品を防ぐため、以下のロジックで発注目標数を自動算出します。

1. **7ヶ月消化モデルの適用**
   - 2026年7月〜2027年1月（7ヶ月間）の消化を前提とするため、過去2年間のデータから**7月〜翌年1月の売上実績のみ**を抽出・集計します。
2. **売れ筋商品の安全バッファ（20%増）**
   - 在庫不足が深刻な定番オリジナル商品（**シレトコ野帳** および **知床オリジナル エコバッグ**）についてのみ、目標数に **1.2倍** のマルチプライヤーを乗算し、四捨五入します。
   - ボトルなどの他商品はバッファを適用せず、等倍（1.0倍）で算出します。
3. **単価重複の統合処理**
   - セールや値引きなどによって過去データで同一SKUに対して異なる単価が存在する場合、自動的に最新の「基準単価（最高単価）」に名寄せし、**1商品1行**に集計をまとめます。
4. **取引先在庫に基づく発注制限（引き当てキャップ）**
   - `確定発注数` ＝ `min(店舗必要数, 取引先在庫数)`
   - 取引先に在庫がない、または廃盤となった商品については、発注数（確保数）が自動的に `0` になり、不足分が `取引先欠品数` として可視化されます。

## 🚀 ツールの実行方法

### 1. 発注計画Excelの生成・更新
コマンドプロンプトやPowerShellで以下を実行します。

```bash
python prepare_order.py --brand Shiretoko --logic average --inventory "在庫一覧 .csv"
```

* **主要なオプション**:
  - `--logic`: 目標数の計算方法（`average`: 2年平均（推奨）, `this_year`: 今年度実績ベース, `last_year`: 昨年度実績ベース）
  - `--wholesale_ratio`: 仕入掛率（デフォルト: `0.8`）
  - `--inventory`: 店舗在庫CSVのパス（省略すると在庫引き当てなしで算出）
  - `--partner_inventory`: 取引先在庫Excelのパス（デフォルト: `知床財団在庫表260708.xlsx`）

*※実行後、`Shiretoko_発注計画.xlsx` が上書き生成されます。*

---

### 2. シレトコ分析Excelへの「品番数比較」シート反映
昨年度と今年度の取り扱い品番・実績・自社在庫・取引先在庫を比較するシートを更新します。

```bash
python add_sku_comparison_sheet.py
```

*※実行後、`シレトコ分析.xlsx` 内に「品番数比較」という名前のシートが作成/上書きされます。*

---

## 🎨 デザインガイドライン (Excelスタイリング)
生成されるExcelファイルは、知床財団のイメージに調和するよう、以下のフォレストグリーンテーマが自動適用されます。
* **フォント**: `BIZ UDPゴシック`（日本語が見やすく潰れにくいフォント）
* **ヘッダー行**: 深緑背景（`#1B5E20`）に白文字
* **合計行**: 非常に薄い緑背景（`#E8F5E9`）に太字。フィルターに自動連動する `SUBTOTAL` 関数を2行目に配置
* **データ明細**: 1行おきにゼブラ柄（極薄のグレー `#F9F9F9`）
* **枠線**: グリッド線を明示的に有効化
